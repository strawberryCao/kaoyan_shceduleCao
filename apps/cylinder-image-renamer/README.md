# AI 缸号图片整理器

从课表项目中独立出来的轻量 Windows 桌面应用。它使用用户自己的阿里云百炼或 OpenAI 兼容 API，识别图片中的缸号，并按照用户定义的格式批量复制或移动、重命名图片。

## 产品定位

这是生产现场的**辅助命名工具**，不是无人值守判定系统：

- 识别稳定的图片自动生成文件名，不要求逐张确认。
- 图片无法可靠读取、返回多个候选或存在实质字符冲突时，才进入手动填写队列。
- 不使用数据库、二维码、本地大模型或用户电脑作为服务器。
- 面向 4–6 GB 内存的 Windows 电脑，任务按顺序逐张处理，原图不会批量驻留内存。

## 默认模型组合

- 首次识别：`qwen3.7-plus`
- 独立复核：`qwen-vl-ocr`
- 加强复核：`qwen3.7-plus`

以上只是内置推荐值，API 地址、三个模型、Temperature、最大输出长度、超时、上传分辨率和全部提示词都可以修改。生产使用时建议在真实样本上验证后固定模型名称或快照版本。

## 可靠性规则

1. 首次视觉识别。
2. 使用另一模型和另一提示词独立复核。
3. 两次结果先执行大小写、空格、分隔符和数字补齐标准化，再进行正则校验。
4. 两次均明确、合法且标准化后完全一致时，自动通过。
5. 其他情况调用第三次加强复核。
6. 如果前两次已经产生两个不同的合法字符结果，系统认定为实质字符冲突，第三次结果只作为参考，仍要求用户填写。
7. 如果前两次只有一个明确结果，另一遍无法读取或不完整，且第三次与明确结果一致，可以标记为“复核通过”。
8. AI 自报的 `confidence` 不参与自动放行。

没有数据库、二维码或工单数据时，系统无法证明一个合法但识别错误的编号一定错误。因此本工具的设计目标是减少确认量，同时拒绝明显不稳定的结果，而不是承诺零漏检。

## 命名规则

默认模板：

```text
{{缸号}}_{{序号}}
```

支持变量：

```text
{{缸号}}
{{序号}}
{{日期}}
{{原文件名}}
{{批次}}
{{扩展名}}
```

还可以配置：

- 缸号正则表达式
- 字母转大写
- 删除空格
- 分隔符统一
- 尾部数字补齐
- 序号起始值和位数
- 批次名称
- 复制或移动模式

同名文件不会被覆盖，程序会自动添加 `(2)`、`(3)` 等后缀。

## API Key

API Key 不写入导出的配置 JSON，也不保存在普通配置文件中。桌面端通过 `keyring` 写入 Windows Credential Manager。

## 开发

要求：

- Node.js 22+
- Rust stable
- Windows WebView2

```powershell
cd apps/cylinder-image-renamer
npm install
npm run tauri dev
```

构建 NSIS 安装文件：

```powershell
npm run tauri build
```

输出位置：

```text
src-tauri/target/release/bundle/nsis/*-setup.exe
```

## 当前原型范围

已实现：

- 批量选择图片和顺序处理
- 千问/OpenAI 兼容视觉请求
- 两次独立识别和第三次加强复核
- 实质字符冲突强制手填
- 自定义 API、模型、参数和提示词
- 自定义缸号规则和文件名模板
- 复制/移动与防覆盖
- 最近一次文件操作撤销
- 配置导入导出，默认不包含 API Key
- 按需加载图片预览，避免大量缩略图常驻内存

尚需用真实生产图片完成：

- 识别正确率与人工填写率验收
- 不同型号铭牌的默认提示词校准
- Windows 代码签名和正式图标
